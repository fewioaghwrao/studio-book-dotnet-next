# Studio Book 画面一覧

## 1. 文書概要

本書は、Studio Book の Next.js フロントエンド実装を基に、現在存在する画面、利用者区分、主要機能、利用APIおよび共通画面部品を整理した棚卸し資料である。

要件定義書、基本設計書、詳細設計書、画面遷移図を作成する際の基礎資料として使用する。

---

## 2. 画面区分

| 区分 | 内容 |
|---|---|
| 共通・公開画面 | 未認証ユーザーを含む全利用者が閲覧できる画面 |
| 認証画面 | ログイン、会員登録、メール認証、パスワード再設定 |
| 一般ユーザー画面 | GeneralUser向けの会員情報、予約、レビュー |
| ホスト画面 | Host向けのスタジオ、予約、レビュー、売上、統計管理 |
| 管理者画面 | Admin向けの会員、スタジオ、予約、ログ、設定管理 |
| システム画面 | エラー、Not Found、ローディング |
| 共通部品 | ヘッダー、フッター、認証ナビゲーションなど |

| 画面分類 | 条件 |

| 一般ユーザー画面 | JWT認証が必要。予約画面はGeneralUserロールが必要 |
| ホスト画面 | JWT認証およびHostロールが必要 |
| 管理者画面 | JWT認証およびAdminロールが必要 |

# 14. 画面実装上の補足事項

1. ログイン後のJWTは `localStorage` の `token` に保存される。
2. 共通認証ナビゲーションは `GET /api/auth/me` でログイン状態とロールを取得する。
3. ロール判定の優先順位は、Admin、Host、GeneralUserの順である。
4. 401発生時は、多くの画面で `/auth/login` へリダイレクトする。
5. 管理者情報編集画面は、公開デモ環境では更新処理が無効化されている。
6. 管理者スタジオ一覧では、公開デモ環境の誤操作防止のため削除機能を提供していない。
7. スタジオ画像は、公開デモではアップロードではなく既存サンプル画像から選択する。
8. AIスタジオ検索は入力文字数制限とAPIレート制限を持つ。
9. レビュー投稿画面にはAIによる文章補助機能が含まれる。
10. ホスト売上画面ではCSV、売上明細画面ではCSVおよびPDFをダウンロードできる。
11. Stripe決済画面自体は外部サービス画面であり、本システムの画面一覧には含めない。
